# Розрахунок ПДВ по елеваторних послугах МХП

## Структура папок

```
mhp/
  calc_pdv.py              ← скрипт (один для всіх місяців)
  CLAUDE.md                ← ця документація
  prices/
    price_пізні_2025-2026.pdf   ← прайс пізніх зернових (оновлюється раз на рік)
    price_ранні_2025-2026.pdf   ← прайс ранніх зернових (оновлюється раз на рік)
  reports/
    2026-03/
      main_report_march2026.xls        ← вхідний звіт
      main_report_march2026_result.xls ← результат (генерує скрипт)
    2026-04/               ← додати наступного місяця
      main_report_april2026.xls
```

## Що робить цей проєкт

Щомісячний звіт `main_report_МІСЯЦЬ.xls` містить 3 вкладки:
- **зберігання** — зберігання зерна (грн/т/день)
- **сушка** — сушка та очистка (грн/т/%)
- **приймання** — приймання зерна (грн/т)

Скрипт `calc_pdv.py`:
1. Бере з прайс-листів суму ПДВ по кожній послузі/організації/культурі
2. Множить на кількість (Дельта або ФизВес)
3. Записує колонки `ціна ПДВ` та `Сума ПДВ` у кожну вкладку
4. Створює 4-ту вкладку **Зведений звіт**: скільки кожна організація виставить кожному контрагенту
5. Переміщення між філіями (де org ≈ контрагент) виводяться окремим блоком

## Як запустити

```bash
# 1. Створи папку для нового місяця і поклади туди файл звіту
#    reports/2026-04/main_report_april2026.xls

# 2. Запусти скрипт:
python3 calc_pdv.py reports/2026-04/main_report_april2026.xls

# Результат з'явиться автоматично поруч:
# reports/2026-04/main_report_april2026_result.xls
```

## Структура вхідного файлу (важливо при змінах формату)

### Вкладка `зберігання`
| Колонка | Індекс | Вміст |
|---------|--------|-------|
| Организация | 0 | Назва елеватора |
| Контрагент | 6 | Кому зберігаємо |
| Культура | 10 | Кукурудза / Соняшник / Соя / Пшениця |
| Дельта | 19 | Кількість т/день |
| ціна ПДВ | 20 | **заповнює скрипт** |
| Сума ПДВ | 21 | **заповнює скрипт** |

Заголовки: рядок 6 (0-indexed). Дані: з рядка 7.

### Вкладка `сушка`
| Колонка | Індекс | Вміст |
|---------|--------|-------|
| Организация | 0 | Назва елеватора |
| Контрагент | 4 | Кому |
| Реальная номенклатура | 8 | Культура |
| Дельта очистка т% | 17 | Кількість очистки (кг%, ділити на 1000!) |
| Дельта сушка т% | 23 | Кількість сушки (кг%, ділити на 1000!) |
| ціна ПДВ очистка | 24 | **заповнює скрипт** |
| Сума ПДВ очистка | 25 | **заповнює скрипт** |
| ціна ПДВ сушка | 26 | **заповнює скрипт** |
| Сума ПДВ сушка | 27 | **заповнює скрипт** |

Заголовки: рядки 4-5. Дані: з рядка 6.
**Формула:** `Сума очистка = (Дельта очистка / 1000) × ціна ПДВ очистка`
**Формула:** `Сума сушка = (Дельта сушка / 1000) × ціна ПДВ сушка`

### Вкладка `приймання`
| Колонка | Індекс | Вміст |
|---------|--------|-------|
| Организация | 0 | Назва елеватора |
| ОтправительПолучатель | 5 | Контрагент |
| Культура | 11 | Культура |
| ФизВес | 18 | Фізична вага (тони) |
| цінаПДВ | 28 | **заповнює скрипт** |
| Сума ПДВ | 29 | **заповнює скрипт** |

Заголовки: рядок 2. Дані: з рядка 3.
**Формула:** `Сума = ФизВес × ціна ПДВ`.

## Що оновлювати щороку / щомісяця

### Щомісяця — нічого, просто запускай:
```bash
python3 calc_pdv.py reports/main_report_МІСЯЦЬ2026.xls
```

### При зміні прайс-листів (новий маркетинговий рік):
Відкрий `calc_pdv.py` та оновіть словник `PRICES` (рядки ~15-115) і `EARLY_GRAIN_PRICES` (~135-140).
Формат: `ціна_з_ПДВ − ціна_без_ПДВ`.

### При зміні формату звіту (нові колонки):
Оновіть константи колонок у відповідних секціях скрипта.
Шукайте коментарі виду: `# cols: org=0, kontrag=6, ...`

## Переміщення між філіями

Якщо організація та контрагент належать до одного юридичного суб'єкта — вважається внутрішнім переміщенням:
- ВКВК ↔ Вінницька птахофабрика
- Вендичанський ↔ МХП-Агрокряж
- МХП-Андріяшівський ↔ МХП-Урожайна Країна
- Яготинський ↔ МХП-Агро-С
- Городенківський ↔ Перспектив
- Захід-Агро МХП (Воскресинцівський / Львівський / Краснянський) — між собою

Ці рядки виводяться червоним у блоці "ПЕРЕМІЩЕННЯ МІЖ ФІЛІЯМИ" і не входять у основний підсумок.
